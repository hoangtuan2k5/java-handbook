# Testing

## Purpose

Folder này ghi lại testing trong Java: unit vs integration test, JUnit 5, Mockito, TDD, và coverage.

Mục tiêu là dùng test để kiểm chứng behavior và thiết kế, không chỉ tăng số lượng test hoặc coverage percentage.

## Suggested reading order

1. `001-unit-test-vs-integration-test.md`
2. `002-junit5.md`
3. `003-mockito.md`
4. `004-tdd.md`
5. `005-test-coverage.md`

## How to use this folder

- Khi test chậm hoặc flaky, phân biệt unit/integration trước.
- Khi viết test API Java hiện đại, đọc JUnit 5.
- Khi mock quá nhiều, đọc Mockito cùng design boundary notes.
- Khi dùng coverage, đọc nó như signal chứ không phải mục tiêu cuối.
