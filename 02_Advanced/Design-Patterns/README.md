# Java Design Patterns

## Purpose

Folder này gom các design pattern phổ biến trong Java theo 3 nhóm: creational, structural, và behavioral.

Mục tiêu không phải là học thuộc tên pattern, mà là hiểu problem, force, trade-off, và khi nào pattern giúp code rõ hơn thay vì làm thiết kế nặng tay.

## Suggested reading path

1. `Creational/`
2. `Structural/`
3. `Behavioral/`

## Folder map

- `Creational/`: Singleton, Builder, Factory, Abstract Factory.
- `Structural/`: Proxy, Decorator, Adapter, Facade.
- `Behavioral/`: Observer, Strategy, Template Method, Chain of Responsibility.

## How to use this folder

- Đọc pattern khi đang có design problem cụ thể, không đọc như checklist phải nhớ hết.
- Luôn hỏi pattern này giải quyết force nào, đổi lại complexity gì, và có simpler alternative nào không.
- Với Java/Spring, ưu tiên nối pattern với API/framework behavior thật thay vì ví dụ quá textbook.
