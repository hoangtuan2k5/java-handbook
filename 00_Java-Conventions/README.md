# Java Conventions

## Purpose

Folder này dùng để lưu các quy tắc tổ chức và đặt tên đặc trưng cho Java trong vault này.

Mục tiêu của nó không phải là gom mọi thứ vào một chỗ thật dài, mà là tạo ra một handbook dễ tra cứu khi mình:

- tạo package mới
- đặt tên class, interface, enum, record
- chia file và sắp xếp code trong file
- viết exception, logging, Javadoc
- review xem code có “giống Java” hay không

## Suggested reading order

Nếu mới bắt đầu, nên đọc theo thứ tự này:

1. `001-folder-structure.md`
2. `002-module-and-package-boundaries.md`
3. `003-package-naming.md`

Các file còn lại nên dùng như reference khi cần.

## Scope

Những rule trong đây ưu tiên cho:

- Java core learning notes
- Java exercise skeletons
- Java code trong Spring-style project

Nếu một framework có convention riêng mạnh hơn, ví dụ Spring Boot, JPA, hay JUnit, thì nên đọc rule ở đây trước rồi điều chỉnh theo framework sau.
